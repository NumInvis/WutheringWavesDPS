## 二、数据获取方案

### 方案：iTunes RSS API（免费、官方）

**接口格式**：
```
https://itunes.apple.com/{country}/rss/topgrossingapplications/limit=100/json
```

**优点**：
- 完全免费，无需API Key
- 官方接口，稳定可靠
- 支持多地区

**限制**：
- 只能获取前100名
- 无法获取精确排名数字（只能通过列表位置推断）
- 每次请求返回完整榜单

## 三、后端实现

### 3.1 数据模型 (`app/models/app_ranking.py`)

```python
class AppInfo(Base):
    """游戏应用信息"""
    id: str (主键)
    name_cn: str  # 中文名
    name_en: str  # 英文名
    app_id_cn: str  # 中国区App ID
    app_id_jp: str  # 日本区App ID
    app_id_us: str  # 美国区App ID
    app_id_kr: str  # 韩国区App ID
    icon_url: str  # 图标URL
    developer: str  # 开发商

class RankingRecord(Base):
    """排名记录"""
    id: str (主键)
    app_id: str (外键)
    country: str  # 地区代码
    rank: int  # 排名（1-100，100+表示未上榜）
    recorded_at: datetime  # 记录时间
    date: str  # 日期（用于按天聚合）
```

### 3.2 爬虫服务 (`app/services/app_ranking_crawler.py`)

```python
# 核心功能
- fetch_ranking(country: str) -> List[RankingData]
  # 从iTunes API获取指定地区的畅销榜
  
- parse_ranking(data: dict, target_apps: List[str]) -> Dict[str, int]
  # 解析榜单，返回目标游戏的排名
  
- save_ranking_records(records: List[RankingRecord])
  # 保存排名记录到数据库

# 定时任务
- 每30分钟爬取一次四个地区的榜单
- 启动时立即爬取一次
```

### 3.3 API接口 (`app/api/app_ranking.py`)

```python
GET /api/ranking/current
# 获取当前最新排名（所有游戏、所有地区）

GET /api/ranking/history/{app_id}
# 获取指定游戏的历史排名趋势

GET /api/ranking/compare
# 获取多游戏对比数据

GET /api/ranking/apps
# 获取监控的游戏列表
```

## 六、技术细节

### 排名处理逻辑
- 排名1-100：直接从榜单获取
- 排名100+：标记为"未上榜"，显示为 ">100"

### 数据存储策略
- 每30分钟记录一次排名
- 保留最近30天的详细数据
- 聚合历史数据（按天/周/月）

### 错误处理
- API请求失败：重试3次
- 解析失败：记录日志，跳过该条数据
- 网络超时：30秒超时
